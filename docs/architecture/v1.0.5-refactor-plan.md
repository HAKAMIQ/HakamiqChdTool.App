# v1.0.5 refactor plan

Scope: v1.0.5 maintenance only.

This plan exists to keep the maintenance release boring. No new conversion promises, no UI rebuild, no `chdman` behavior changes.

## Refactor targets

- Conversion orchestration: split long-running workflow coordination from low-level `chdman` execution details.
- Path planning: isolate output path, pending workspace, and final promotion rules.
- Queue state transitions: keep cancellation, skipped-existing, failed, and success outcomes as explicit terminal states.
- Cleanup policy: centralize pending workspace deletion and legacy pending-root cleanup behind one policy surface.
- Storage classification: keep `StorageDeviceKind` as the source of truth and map it to UI text only at presentation boundaries.

## Guardrails

- Do not change queue semantics.
- Do not change defaults.
- Do not introduce a new UI framework.
- Do not change release packaging policy.
- Do not add new format support inside a maintenance refactor.

A refactor that changes user-visible conversion behavior is no longer v1.0.5 maintenance work.
